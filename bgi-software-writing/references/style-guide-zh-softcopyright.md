# 中文软著写作风格规范

## 语体要求
- 使用正式、客观、技术性陈述句。
- 避免口语化表达和宣传性语言。
- 同一概念保持单一术语，不频繁换词。

## 句式与段落
- 每段只讲一个核心点。
- 先定义，再说明机制，再给场景与边界。
- 涉及流程时优先采用“步骤化描述+图示”。

## 术语管理
- 首次出现的英文缩写需给中文全称与英文全称。
- 专有名词在全文统一写法（如模块名、字段名）。

## 图注与表注
- 图注格式：`图 n 标题`
- 表注格式：`表 n 标题`
- 图表标题应表达“对象+目的”，避免泛化（如“系统图”）。

## 禁忌
- 大段空泛描述无技术细节。
- 将平台管理模块强行写成“输入输出执行步骤”。
- 主干结构与软件类型不匹配。
